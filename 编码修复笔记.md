# Windows 控制台编码修复笔记

## 现象判断

当前机器检测到：

- `HKCU\Console\CodePage = 65001`
- `HKCU\Console\C:_Windows_System32_cmd.exe\CodePage = 65001`
- `HKLM\SYSTEM\CurrentControlSet\Control\Nls\CodePage\OEMCP = 65001`
- PowerShell 配置文件中也加入了 `chcp 65001` 和 UTF-8 编码设置

系统区域信息显示为 `zh-CN`，这意味着你的中文 Windows 传统控制台兼容编码通常应为：

- ANSI/ACP：通常是 `936`
- OEMCP：通常也是 `936`

`图吧工具箱` 这类偏传统的 Win32 程序，出现乱码的最可疑原因就是你把系统级 `OEMCP` 改成了 `65001`。

## 推荐修复思路

目标是：

1. 恢复系统和 `cmd` 的传统中文兼容编码
2. 保留 PowerShell 的 UTF-8 能力，尽量不影响 Codex

也就是：

- 把 `HKLM ... OEMCP` 从 `65001` 改回 `936`
- 删除或恢复 `HKCU\Console` 下给 `cmd` 写死的 `CodePage`
- 保留 PowerShell 中的 UTF-8 输出/输入设置
- 不再在 PowerShell 启动时强制执行 `chcp 65001`

## 建议执行顺序

### 第一步：管理员身份打开 PowerShell

必须用“管理员身份运行”的 PowerShell 才能改回 `HKLM`。

### 第二步：恢复系统 OEM 代码页

执行：

```powershell
reg add "HKLM\SYSTEM\CurrentControlSet\Control\Nls\CodePage" /v OEMCP /t REG_SZ /d 936 /f
```

这是最关键的一步。

### 第三步：撤销给 cmd 写死的 UTF-8

执行：

```powershell
reg delete "HKCU\Console" /v CodePage /f
reg delete "HKCU\Console\%SystemRoot%_system32_cmd.exe" /v CodePage /f
reg delete "HKCU\Console\C:_Windows_System32_cmd.exe" /v CodePage /f
```

说明：

- 删除比改成固定 `936` 更稳妥
- 删除后，`cmd` 会回到系统默认控制台编码行为

如果某一条提示“找不到指定的注册表项或值”，可以忽略。

### 第四步：清理 PowerShell 配置里不必要的 `chcp 65001`

你当前 `PowerShell Profile` 里大概率有这些行：

```powershell
chcp 65001
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
[Console]::InputEncoding = [System.Text.Encoding]::UTF8
$OutputEncoding = [System.Text.Encoding]::UTF8
```

建议改成只保留下面三行，删除 `chcp 65001`：

```powershell
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
[Console]::InputEncoding = [System.Text.Encoding]::UTF8
$OutputEncoding = [System.Text.Encoding]::UTF8
```

原因：

- `chcp 65001` 会直接改控制台代码页
- 对老程序兼容性影响更大
- 而三条 `.NET` 编码设置更偏向 PowerShell 自身 I/O，通常已经够 Codex/脚本使用

## 可直接用于编辑 Profile 的命令

先查看当前配置文件路径：

```powershell
$PROFILE
```

然后手动把其中的：

```powershell
chcp 65001
```

删掉，保留另外三行 UTF-8 设置。

## 第五步：重启电脑

改完 `OEMCP` 后，建议重启系统再测试。

至少要满足：

- 关闭所有 `cmd` / PowerShell / 图吧工具箱
- 重启 Windows

否则有些程序仍会沿用旧环境。

## 修复后如何验证

### 验证系统 OEMCP

执行：

```powershell
Get-ItemProperty -Path "HKLM:\SYSTEM\CurrentControlSet\Control\Nls\CodePage" | Select-Object OEMCP
```

期望结果：

```text
OEMCP
-----
936
```

### 验证 cmd 当前代码页

打开新的 `cmd`，执行：

```cmd
chcp
```

常见预期是：

```text
活动代码页: 936
```

### 验证 PowerShell 是否仍能正常 UTF-8

打开新的 PowerShell，执行：

```powershell
[Console]::OutputEncoding
```

如果显示 UTF-8 相关信息，一般就说明 PowerShell 侧设置还在。

## 如果你还想兼顾 Codex 中文输出

更推荐的策略是：

- 不改 `HKLM` 的 `OEMCP`
- 不全局改 `cmd` 注册表代码页
- 只在 PowerShell 里保留 UTF-8 编码设置
- 如果某个单独终端需要 UTF-8，再在那个窗口里临时执行 `chcp 65001`

这样对老软件的破坏最小。

## 一键回退命令汇总

下面这组命令建议在“管理员 PowerShell”里逐条执行：

```powershell
reg add "HKLM\SYSTEM\CurrentControlSet\Control\Nls\CodePage" /v OEMCP /t REG_SZ /d 936 /f
reg delete "HKCU\Console" /v CodePage /f
reg delete "HKCU\Console\%SystemRoot%_system32_cmd.exe" /v CodePage /f
reg delete "HKCU\Console\C:_Windows_System32_cmd.exe" /v CodePage /f
```

然后再手动编辑 PowerShell Profile，删除：

```powershell
chcp 65001
```

保留：

```powershell
[Console]::OutputEncoding = [System.Text.Encoding]::UTF8
[Console]::InputEncoding = [System.Text.Encoding]::UTF8
$OutputEncoding = [System.Text.Encoding]::UTF8
```

最后重启系统。
