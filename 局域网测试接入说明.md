## 推荐联调拓扑

```text
浏览器 / 手机 / 平板
        |
        v
   http://宿主机IP:9010
        |
        +--> /              -> Vite dev server (9510)
        +--> /lottery-*     -> Gateway (9008)
        +--> /swagger-ui*   -> Gateway (9008)
        +--> /v3/api-docs*  -> Gateway (9008)
```

这样设计的好处：

- 前端页面和接口都从同一个入口访问
- 前端继续走相对路径，不需要在代码里硬编码后端地址
- 后端服务仍然保留独立端口，便于逐个排查
- Jenkins 也能独立访问，适合你边开发边练 CI

## 启动顺序

### 1. 启动 Docker 基础设施

```powershell
cd C:\develop\project\lottery-platform\lottery-platform-backend\deploy\docker
.\scripts\start-all.ps1
```

如果你暂时不想启动整套环境，只想起 Jenkins：

```powershell
cd C:\develop\project\lottery-platform\lottery-platform-backend\deploy\docker
.\scripts\start-jenkins.ps1
```

### 2. 启动后端服务

```powershell
cd C:\develop\project\lottery-platform\lottery-platform-backend
mvn -pl lottery-gateway -am spring-boot:run
mvn -pl lottery-user -am spring-boot:run
mvn -pl lottery-activity -am spring-boot:run
mvn -pl lottery-lottery -am spring-boot:run
mvn -pl lottery-award -am spring-boot:run
mvn -pl lottery-pay -am spring-boot:run
mvn -pl lottery-workflow -am spring-boot:run
mvn -pl lottery-file -am spring-boot:run
mvn -pl lottery-monitor -am spring-boot:run
```

### 3. 启动前端

```powershell
cd C:\develop\project\lottery-platform\lottery-platform-frontend
Copy-Item .env.example .env -ErrorAction SilentlyContinue
npm install
npm run dev
```

## 内网穿透建议

如果只是临时给外网设备做演示或真机测试，我更推荐：

- `natapp`：接入快，适合临时演示
- `frp`：如果你有自己的服务器，更稳定也更可控

### natapp 用法

建议直接把公网入口映射到统一测试入口 `9010`：

```powershell
natapp -authtoken=<你的Token> -http 9010
```

这样测试人员访问一个公网地址就够了，不需要关心内部微服务端口。

### frp 思路

如果你使用 `frpc`，建议本地映射目标设置为：

- 本地 IP：当前开发机在局域网中的 IP
- 本地端口：`9010`

不建议直接暴露 `9008`、`9101` 这种内部调试端口。

## 注意事项

- `nginx` 通过 `host.docker.internal` 回源到宿主机上的 `9510` 和 `9008`，所以前端和网关需要在宿主机上启动
- Jenkins 当前默认管理员账号为 `admin / 123456`
- 如果只是局域网内测试，其实不需要内网穿透，直接访问 `http://宿主机IP:9010` 即可
- 如果 Windows 防火墙拦截了 `9010`、`9510`、`9008`、`9200`，需要放开这些端口
- 当前方案是测试项目方案，不建议直接照搬到生产环境
